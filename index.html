Biography

He is currently working as a senior engineer at Commvault Systems Inc. in New Jersey. He received his bachealor's degree in 2013 from VIT University. Before comming to NCSU, he had been a part of Commvault Systems as a senior engineer where his responsibilities included designing components and training freshers. Projects
  • Voice Based - Bot
    An application for integration of CommVault with Google Home and Amazon Alexa. This app allowed user to interact using natural voice conversation for retrieve certain health status details such as SLA value, Anomaly report etc. For authentication, open source project “MITRE-OPENID” was extended to support OAUTH protocol. Business Login was implemented in JAVA Spring and was invoked from Amazon/Google service end-point via REST API.
    Ideal for : Quick and natural interactions with software to solve problems
  • Open vSwitch
    A Project to replace the existing KVM Linux bridge with Open vSwitch on Virtual Computing Lab to add programmability into the network. This project was conducted in collaboration with NCSU’s VCL team and in a team of 4 fellow students. This project was a prototype to evaluate the technology before migrating into production.
    Ideal for : Implementing a flexible and programmable network in a datacenter
    Source: https://git.io/v55Ib
  • ChordX
    A Research project to improve chord algorithm in a distributed dictionary. Chord is well known algorithm to identify nodes containing resources in peer to peer network without any central server. I was responsible to add redundancy in the algorithm to improve the lookup query time.
    Ideal for : An unorganized distributed system with fault tolerance
    Source: https://goo.gl/UuKcz3
  • RAM FUSE Filesystem
    A filesystem that stores all the files and directories in memory instead of disk blocks. Persistence is supported by dumping all the metadata and data onto a file in disk.
    Ideal for : Frequently used file with less open time
    Not recommended for : Large files due to high memory consumption
    Source: https://github.com/dgupta9/
  • Thread Library
    A non-preemptive thread library for linux OS. The library uses the concept of context switching (via ucontext.h) to implement user level thread. It also include support for thread synchronization by its own implementation of semaphores.
    Advantages: light weight library with minimal system requirement
    Source: https://github.com/dgupta9/
  • Peer to Peer chat application
    Application that implements concepts of distributed systems to allow clients comunicate without the need of central server. Issues such as Single point of failure were eliminated by using the concept of multiple trackers. Custom application and network layer protocols are defined to allow client-to-client or client-tracker communications.
    Source: https://github.com/dgupta9/
  • Simple DB Buffer and Transaction
    Improved version of SimpleDB [ Link ]. The default buffer pool management is replaced with FIFO(First In First Out) policy. Also the recovery manager was enhanced from 'Undo Algorithm' to 'Undo-Redo Algorithm'.
    Source: https://github.com/dgupta9/
